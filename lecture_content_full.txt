PARA: 1. What is Agentic AI?
PARA: Agentic AI refers to artificial intelligence systems that can act autonomously toward a goal — planning, executing actions, using tools, and adapting based on feedback — without requiring step-by-step human instructions. Unlike a standard chatbot that responds to a single prompt and stops, an AI Agent can reason across multiple steps, call external tools, remember intermediate results, and coordinate with other agents to solve complex tasks.
PARA: 1.1  The Four Pillars of an AI Agent
PARA: 1.2  Why Agentic AI Matters for Linguistics
PARA: Traditional NLP pipelines are rigid: each step is hardcoded. Agentic systems are adaptive and modular: each agent can re-evaluate its strategy, handle exceptions, and pass enriched data downstream — exactly what large-scale linguistic corpus analysis demands.
PARA: Handles diverse file formats: .txt, .csv, .json, PDF corpora
PARA: Applies multiple linguistic frameworks simultaneously across layers
PARA: Produces structured, evidence-based linguistic reports automatically
PARA: Scales from a single essay to a million-token corpus
PARA: 2. How AI Agents Work — Core Architecture
PARA: Every AI agent follows the Observe -> Think -> Act -> Reflect cycle. Understanding this loop is essential before designing any multi-agent linguistics system.
PARA: 2.1  Types of Agent Memory
PARA: 2.2  Agent Communication Patterns
PARA: Sequential (Pipeline): Agent 1 -> Agent 2 -> Agent 3 -> Final Output. Each agent hands its result to the next. Simple and predictable.
PARA: Hierarchical (Orchestrator + Sub-agents): A master agent delegates tasks to specialist sub-agents and synthesizes their results.
PARA: Parallel (Swarm): Multiple agents work simultaneously on different parts of the data, and their outputs are merged.
PARA: In our PMDD project, we use a hybrid pattern: sequential pipeline at the macro level (agents 1 to 5) with an orchestrator (Agent 5) that can request re-analysis from any earlier agent if evidence is insufficient.

PARA: 4. The New Project: Pragmatic Meaning Drift Detector (PMDD)
PARA: 4.1  What Does PMDD Analyze?
PARA: PMDD investigates how the pragmatic meaning, connotative associations, and discourse functions of key lexical items shift across different time periods, genres, or social groups within a text corpus. This is a core question in Historical Pragmatics, Sociolinguistics, and Corpus Linguistics.
PARA: Examples of research questions PMDD can answer from a large corpus:
PARA: How has the word "community" shifted in its pragmatic use across political speeches from 2000 to 2024?
PARA: Do academic research papers in 2010 use hedging language differently from those in 2024?
PARA: How do informal social media texts differ from formal news texts in their use of causal connectives?
PARA: Is there evidence of register borrowing — formal vocabulary appearing in informal contexts?
PARA: How do politeness strategies evolve across different sections of a legal document corpus?

Key Distinction: Passive AI vs Agentic AI
Passive AI (e.g., plain GPT-4 chat): You ask a question and it answers. The process ends there.
Agentic AI: You give a high-level goal. It plans sub-tasks, executes tools, evaluates results, and iterates.
In linguistics terms: Passive AI = single utterance response. Agentic AI = full discourse with tool-assisted reasoning.

Pillar | Description
Perception | The agent receives input — text, files, structured data, API results, or output from other agents.
Reasoning | Using a Large Language Model (LLM), the agent interprets the input, plans actions, and decides what tool to call next.

Memory | The agent stores context within a session (short-term) or retrieves results from persistent storage (long-term).

TABLE START
The Agent Loop (Observe -> Think -> Act -> Reflect)
OBSERVE:   The agent receives data — raw text, preprocessed output, or another agent's result.
THINK:     The LLM reasons about what to do next (often using chain-of-thought prompting).
ACT:       The agent executes: calls a tool, writes code, calls the OpenAI API, or delegates.
REFLECT:   The agent evaluates its own output and decides to continue or stop.
This loop repeats until the goal is achieved or a stopping condition is met.
TABLE END

Memory Type | Description
In-Context Memory | Information stored within the current LLM context window. Temporary — lost when the session ends.
External Memory | Information stored in files, databases, or vector stores. Persists across sessions.
Episodic Memory | Records of past agent actions and results. Enables agents to learn from previous runs.
Semantic Memory | Embedded representations of knowledge (e.g., embeddings of a corpus for similarity search).

Theory | Application in PMDD
Pragmatics — Grice (1975) | Cooperative Principle: Maxims of Quantity, Quality, Relation, Manner. Implicature patterns indicate how meaning shifts beyond literal semantics.
Speech Act Theory — Austin/Searle | Illocutionary force classification: assertives, directives, commissives, expressives, declarations. Agent 2 classifies speech acts per segment.
Semantic Field Theory — Lyons (1977) | Lexical items organized into semantic fields. Drift is detected when a word moves between fields or acquires new field membership over time.
Register Analysis — Halliday (1978) | Tenor, Field, Mode. Agent 3 identifies register markers and detects register borrowing or shift across corpus sections.
Corpus Linguistics — Sinclair (1991) | Concordance, collocation, frequency, keyness. Agent 4 performs quantitative corpus analysis to detect statistically significant lexical patterns.
Critical Discourse Analysis — Fairclough | Power, ideology, and social context encoded in text. Agent 5 synthesizes all layers into a critical evidence report.

# agents/agent1_preprocessor.py
import spacy, json, re
nlp = spacy.load('en_core_web_sm')

def run_agent1(file_path: str) -> dict:
with open(file_path, 'r', encoding='utf-8') as f:
raw_text = f.read()
text = re.sub(r'\s+', ' ', raw_text).strip()
doc = nlp(text)
segments = []
for i, sent in enumerate(doc.sents):
segments.append({
'id': i,
'text': sent.text.strip(),
'word_count': len([t for t in sent if not t.is_space]),
'position': round(i / len(list(doc.sents)), 2)
})
return {
'corpus_stats': {'total_sentences': len(segments),
'total_words': sum(s['word_count'] for s in segments)},
'segments': segments
}

# agents/agent2_pragmatic.py
from openai import OpenAI
client = OpenAI(api_key='YOUR_OPENAI_API_KEY')

SYSTEM_PROMPT = '''You are a computational linguistics expert.
Analyze the given text segment using these exact frameworks:
1. Speech Act Theory (Searle): classify as one of:
Assertive / Directive / Commissive / Expressive / Declaration
2. Gricean Maxims: list any violations:
Quantity (too much/too little info) / Quality (unverifiable claim) /
Relation (irrelevant) / Manner (unclear/ambiguous)
3. Implicature: identify as conventional or conversational
4. Politeness: score 1-5 (1=very face-threatening, 5=highly polite)
Return ONLY valid JSON with keys:
{speech_act, maxim_violations[], implicature, politeness_score}'''

def analyze_segment(segment_text: str) -> dict:
response = client.chat.completions.create(
model='gpt-4o',
messages=[{'role':'system','content':SYSTEM_PROMPT},
{'role':'user','content':f'Analyze: {segment_text}'}],
temperature=0.2
)
import json
return json.loads(response.choices[0].message.content)

def run_agent2(segments: list) -> list:
results = []
for seg in segments:
analysis = analyze_segment(seg['text'])
results.append({**seg, **analysis})
return results

# agents/agent5_orchestrator.py
def run_orchestrator(a1_out, a2_out, a3_out, a4_out, client):
# Quality check: if < 50% segments have speech act labels, re-run Agent 2
labeled = sum(1 for s in a2_out if s.get('speech_act'))
if labeled / len(a2_out) < 0.5:
print('Agent 2 coverage low — requesting re-analysis...')
from agents.agent2_pragmatic import run_agent2
a2_out = run_agent2(a1_out['segments'])

synthesis_prompt = f'''
You are a senior computational linguist. Synthesize these findings:
- Corpus stats: {a1_out['corpus_stats']}
- Pragmatic sample: {a2_out[:5]}
- Semantic drift: {a3_out}
- Corpus statistics: {a4_out}

Write a formal linguistic evidence report with these sections:
1. Executive Summary (2 paragraphs)
2. Pragmatic Drift Evidence (cite segment IDs)
3. Semantic Field Shifts (cite specific vocabulary)
4. Register Analysis Findings
5. Corpus Statistics Summary
6. Overall Meaning Drift Score (0-100) with justification
'''
response = client.chat.completions.create(
model='gpt-4o',
messages=[{'role':'user','content':synthesis_prompt}],
max_tokens=2000
)
return response.choices[0].message.content

# app.py
import gradio as gr
from openai import OpenAI
from agents.agent1_preprocessor  import run_agent1
from agents.agent2_pragmatic     import run_agent2
from agents.agent3_semantic      import run_agent3
from agents.agent4_statistics    import run_agent4
from agents.agent5_orchestrator  import run_orchestrator

client = OpenAI(api_key='YOUR_OPENAI_API_KEY')

def analyze_corpus(file_obj, target_keywords):
keywords = [k.strip() for k in target_keywords.split(',')]
log = []
log.append('Running Agent 1: Preprocessing...')
a1 = run_agent1(file_obj.name)
log.append(f'Agent 1 complete. Segments: {len(a1["segments"])}')
a2 = run_agent2(a1['segments'])
log.append('Agent 2 complete: Pragmatic analysis done.')
a3 = run_agent3(a1['segments'], keywords, client)
log.append('Agent 3 complete: Semantic drift detected.')
a4 = run_agent4(a1['segments'])
log.append('Agent 4 complete: Corpus statistics ready.')
report = run_orchestrator(a1, a2, a3, a4, client)
log.append('Agent 5 complete. Report generated!')
return '\n'.join(log), report

demo = gr.Interface(
fn=analyze_corpus,
inputs=[
gr.File(label='Upload Corpus (.txt or .csv)'),
gr.Textbox(label='Target Keywords (comma-separated)',
value='community, power, change')
],
outputs=[
gr.Textbox(label='Agent Progress Log'),
gr.Textbox(label='Linguistic Report', lines=30)
],
title='PMDD — Pragmatic Meaning Drift Detector',
description='Upload a text corpus and receive a full linguistic analysis report.'
)
demo.launch()
